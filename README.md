# NexusBI v6 — Enterprise Business Intelligence Platform

Full-stack BI platform: Next.js 15 + Cube.js + Oracle + React 19 + NextAuth.js v5.
Runs entirely with `npm` — no Docker, no Redis, no MinIO, no DuckDB, no OpenSSL.

---

## What's in the box

- **27 database connectors** (Postgres, MySQL, Oracle, SQL Server, Snowflake, BigQuery, Redshift, Databricks, Athena, Hive, Spark, Impala, Presto, Trino, ClickHouse, MongoDB, plus CSV/Excel/Parquet/JSON files and REST/GraphQL/Webhook/OAuth APIs)
- **Workbook system** — Tableau-equivalent personal workspace: workbooks, worksheets, calculated fields, parameters, stories
- **Dashboard builder** with drag-and-drop grid, NL filter bar, dual-checker approval workflow
- **6-role RBAC** (PLATFORM_ADMIN → DS_ADMIN → AUTHOR → CHECKER → VIEWER → EMBED_ONLY)
- **Approval replay**, performance scorecard, smart pre-aggregations, outbound webhooks, timed access grants
- **41 Oracle tables**, 46 permissions, 60+ API endpoints — all audited

## First-Time Setup

### Step 1 — Clone and install

```bash
git clone <repo-url> nexusbi
cd nexusbi
npm install
```

### Step 2 — Create the schema user in Oracle

In **SQL Developer** as SYSTEM:

```sql
CREATE USER nexusbi IDENTIFIED BY NexusBIPwd123;
GRANT CONNECT, RESOURCE, CREATE SESSION, CREATE TABLE, CREATE SEQUENCE,
      CREATE PROCEDURE, CREATE TRIGGER, CREATE VIEW TO nexusbi;
ALTER USER nexusbi QUOTA UNLIMITED ON USERS;
```

### Step 3 — Run the seed script

In SQL Developer, connect as the `nexusbi` user and run the entire contents of `oracle-seed.sql`. This creates:

- 41 tables (31 core + 10 workbook tables)
- 46 permissions (38 core + 8 workbook/story)
- 6 roles + permission grants
- 6 dummy users for testing
- 3 sample dashboards (1 APPROVED, 1 DRAFT, 1 PENDING_REVIEW)
- 1 sample workbook with 2 worksheets, 2 calculated fields, 2 parameters
- 1 sample story
- Dashboard share grants (so VIEWER role sees only specifically-shared dashboards)

Verify the seed worked:

```sql
SELECT COUNT(*) FROM NB_PERMISSIONS;       -- 46
SELECT COUNT(*) FROM NB_ROLES;             -- 6
SELECT COUNT(*) FROM NB_USERS;             -- 6
SELECT COUNT(*) FROM NB_DASHBOARDS;        -- 3
SELECT COUNT(*) FROM NB_WORKBOOKS;         -- 1
SELECT COUNT(*) FROM NB_WORKSHEETS;        -- 2
SELECT COUNT(*) FROM NB_DASHBOARD_SHARES;  -- 2
```

### Step 4 — Generate secrets (no OpenSSL needed)

```bash
npm run generate:secrets
```

This runs `node scripts/generate-secrets.js` and prints four secrets generated by Node.js's built-in `crypto.randomBytes`. **No OpenSSL, no external tools.**

Output looks like:

```
NEXTAUTH_SECRET=<48 random bytes, base64>
ENCRYPTION_KEY=<32 random bytes, hex>
CUBEJS_API_SECRET=<32 random bytes, base64>
INTERNAL_API_SECRET=<32 random bytes, base64>
```

### Step 5 — Create `apps/web/.env.local`

```bash
cp .env.example apps/web/.env.local
```

Edit `apps/web/.env.local` and paste your generated secrets. The Oracle connection uses the **same fields from SQL Developer**:

```env
ORACLE_HOST=your-db-hostname
ORACLE_PORT=1521
ORACLE_SERVICE=XEPDB1
ORACLE_USER=nexusbi
ORACLE_PASSWORD=NexusBIPwd123

NEXTAUTH_SECRET=<paste>
NEXTAUTH_URL=http://localhost:3000
ENCRYPTION_KEY=<paste-64-hex-chars>
CUBEJS_API_SECRET=<paste>
CUBEJS_API_URL=http://localhost:4000
INTERNAL_API_SECRET=<paste>
INTERNAL_API_URL=http://localhost:3000
```

### Step 6 — Create local storage directories

```bash
npm run storage:init
```

Creates `./data/storage/` with subdirectories for file uploads, Tableau imports, exports, and templates. (Replaces MinIO / S3.)

### Step 7 — Start the application

```bash
npm run dev
```

Two processes start concurrently:

| Process | Port | Purpose |
|---------|------|---------|
| **Next.js** | 3000 | Frontend + API routes |
| **Cube.js** | 4000 | Query orchestration + caching |

### Step 8 — Open the app and log in

Go to http://localhost:3000/login

| Username | Password | Role |
|----------|----------|------|
| `admin` | `Admin@123!` | PLATFORM_ADMIN |
| `author1` | `Test@123!` | AUTHOR (the Maker — has workbooks) |
| `checker1` | `Test@123!` | CHECKER |
| `checker2` | `Test@123!` | CHECKER (for dual-checker tests) |
| `viewer1` | `Test@123!` | VIEWER (sees ONLY shared dashboards) |

## Workbook System (Tableau Replacement)

The Maker (AUTHOR role) works in **workbooks** — personal workspaces analogous to Tableau's `.twbx` files.

| Tableau concept | NexusBI equivalent | Where it lives |
|-----------------|-------------------|----------------|
| Workbook (.twbx) | **Workbook** | `NB_WORKBOOKS` table, `/workbooks/[id]/edit` |
| Worksheet (chart sheet) | **Worksheet** | `NB_WORKSHEETS` table, Worksheet tab in editor |
| Calculated Field | **Calculated Field** | `NB_CALCULATED_FIELDS` table, Calc Fields tab |
| Parameter | **Parameter** | `NB_PARAMETERS` table, Parameters tab |
| Story | **Story** | `NB_STORIES` + `NB_STORY_POINTS`, `/stories/[id]` |
| Dashboard Action | **Dashboard Action** | `NB_DASHBOARD_ACTIONS` table |
| Sets | **Data Set** | `NB_DATA_SETS` table |
| Groups | **Data Group** | `NB_DATA_GROUPS` table |
| Data Extract | **Data Extract** | `NB_DATA_EXTRACTS` table → Oracle staging |
| Marks card (color/size/label/tooltip) | **Worksheet Shelves** | `SHELVES_JSON` column |
| Filters shelf | **Worksheet Filters** | `WorksheetFilter[]` in shelves |
| Reference lines | **Reference Lines** | `REF_LINES_JSON` column |
| Annotations | **Annotations** | `ANNOTATIONS_JSON` column |
| Publish to Server | **Publish Workbook** | `POST /api/workbooks/[id]/publish` → DRAFT dashboard |

### Workflow

1. AUTHOR creates a workbook (`/workbooks` → New Workbook)
2. Adds worksheets, drags fields onto Columns / Rows / Color / Size shelves
3. Defines calculated fields (custom SQL expressions per workbook)
4. Defines parameters (user-input variables: string / integer / float / date / boolean with all / list / range allowables)
5. Clicks **Publish** → workbook becomes a DRAFT dashboard
6. Submits dashboard for CHECKER review
7. Two CHECKERs approve → APPROVED → visible to VIEWERs (only if they're in the share list)

### Stories

Stories are multi-frame narratives: each "story point" is a dashboard, worksheet, text slide, or image. AUTHORs create stories in `/stories`, add points via the filmstrip UI, then publish.

## File / API Source Ingest (Oracle Staging — replaces DuckDB)

Files (CSV/Excel/JSON) and API sources (REST/GraphQL/Webhook/OAuth) used to be ingested into DuckDB. **They now ingest into Oracle staging tables**:

- File upload → parsed → `CREATE TABLE NB_STG_<sanitized_id> (...)` in Oracle → batch INSERT
- API source poll → fetched → `CREATE TABLE NB_STG_API_<sanitized_id> (...)` in Oracle → batch INSERT
- Cube.js queries these staging tables via the Oracle driver

**Type mapping (during ingest):**

| Inferred type | Oracle column type |
|---------------|-------------------|
| Integer | `NUMBER(19)` |
| Float / decimal | `NUMBER(38,10)` |
| Boolean | `NUMBER(1)` |
| Date / timestamp | `TIMESTAMP` |
| String ≤ 200 chars | `VARCHAR2(500)` |
| String ≤ 2000 chars | `VARCHAR2(4000)` |
| Long string | `CLOB` |

If you'd prefer Hive or MongoDB as the staging layer instead of Oracle, swap `apps/web/lib/file-ingest.ts` and `apps/web/lib/api-source-ingest.ts` to use those drivers instead.

## Viewer Access (Important — read this if VIEWER seems "stuck")

A VIEWER **cannot** see all approved dashboards. They see ONLY:

1. Dashboards they **own**, OR
2. Dashboards explicitly shared with them via `NB_DASHBOARD_SHARES` (per-user OR per-role grant) AND the dashboard is APPROVED

The seed script grants `viewer1` and the entire VIEWER role access to the "Q4 Revenue Overview" dashboard, so they have one visible dashboard out of the box.

To grant a viewer access to another dashboard:

```sql
INSERT INTO NB_DASHBOARD_SHARES (SHARE_ID, DASH_ID, USER_ID, CAN_EDIT, CAN_SHARE, ORG_ID)
  VALUES (SYS_GUID(), 'DASH_XXX', 'USR_VIEWER1', 0, 0, 'ORG_DEFAULT');
```

Or share with the entire VIEWER role:

```sql
INSERT INTO NB_DASHBOARD_SHARES (SHARE_ID, DASH_ID, ROLE_ID, CAN_EDIT, CAN_SHARE, ORG_ID)
  VALUES (SYS_GUID(), 'DASH_XXX', 'R_VIEWER', 0, 0, 'ORG_DEFAULT');
```

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `ORACLE_HOST is required` | Set ORACLE_HOST/PORT/SERVICE/USER/PASSWORD in .env.local |
| `ORA-12514` | Wrong service name — use Service Name from SQL Developer (NOT SID) |
| `ORA-01017` | Wrong username or password |
| Login fails with correct password | Seed script didn't run — execute oracle-seed.sql as nexusbi user |
| `Cube.js not reachable` | Check port 4000 — `npm run dev` starts both processes |
| File upload fails | Run `npm run storage:init` |
| `ENCRYPTION_KEY must be 32 bytes` | Run `npm run generate:secrets` and paste fresh values |
| VIEWER sees no dashboards | They need explicit shares — see "Viewer Access" section above |
| `duckdb` errors | Make sure you ran `npm install` after the v6 update — DuckDB has been removed |
